사이드 프로젝트

문제

시험 기간마다 기출문제를 찾고 풀이를 확인하는 것이 불편해 직접 개발. 직접 크롤링으로 106,495건 적재 후 API 응답이 1,596ms로 급격히 저하. EXPLAIN ANALYZE 분석 결과 exam_id 인덱스 누락으로 전체 Sequential Scan 발생 확인.

문제 해결 과정
  • DB 인덱스 설계·추가 (exam_id, subject_id, 복합 인덱스) → Sequential Scan 제거
  • API 응답 여전히 수백ms → 추가 분석으로 병목의 95.7%가 네트워크(로컬→원격DB)임을 수치로 규명
  • SELECT 컬럼 최소화, 쿼리 통합(2회→1회), Connection Pool 설정, COUNT 캐싱 단계별 적용
  • 프론트가 문제를 1개씩 표시하는데 백엔드가 전체 로드 중임을 발견 → 페이지네이션 추가
성과
  • API 응답 1,596ms → 273ms (83% 개선)
  • Python 크롤링으로 106,495건 문제 직접 수집·적재
  • JWT 인증, 과목·시험·문제 CRUD REST API 개발 및 AWS EC2 배포
  • Cloudflare R2 이미지 저장 및 URL 기반 서빙 구조 적용
  • 시험지 PDF 파싱 기능 개발 중 진행 중
