아키아카

블록체인 전담 인력 없는 2인 팀에서 기술 검토·스택 선정·구현 방향 수립 주도. 백엔드 API, 스마트컨트랙트 연동, 배포 이슈 대응까지 실질적 구현 담당.
기술 선택
  • 이더리움 메인넷 → 트랜잭션 비용 발생, 공공기관 예산 구조 부적합
  • Hyperledger Fabric → EVM 비호환, 향후 확장성 제한
  • Hyperledger Besu 선택 — 비용 없는 프라이빗 환경 + EVM 호환으로 확장 가능
  • 외부 스토리지(Pinata 등) → 공공기관 외부망 차단으로 불가 → Private IPFS 직접 구축
  • HTTPS→HTTP Mixed Content 에러 → 백엔드·IPFS 경로를 Apache ProxyPass로 통합해 HTTPS 단일 경로로 해결
성과 — 양천구

  • Hyperledger Caliper로 네트워크 TPS·레이턴시 성능 한계 사전 파악
